## [v0.1.0] - 2026-04-02

### 新增功能 (Added)
* 新增 Android 端导入/导出工作流：支持通过 SAF 选择目录执行 `Import TXT Folder`、选择 zip 执行 `Import Archive`、创建 zip 执行 `Export Archive`。
* 新增 Native 桥接能力：增加 `rebuildFromLogs`、`exportArchive`、`importArchive`、`queryCycles`、`queryCycleVolumes`、`queryCycleTypeReport` JNI 接口与返回 payload 组装。
* 新增周期维度查询与报告预览：在 Query 页面新增 `cycles` 展开、按动作类型展示 `volume stats`、按 `cycle/type` 懒加载 Markdown 报告。
* 新增显示单位配置：在 Config 页面增加 `Original`/`KG`/`LB` 选项，并通过 DataStore `display_unit` 持久化。
* 新增显示单位格式化模块：新增 `DisplayUnitFormatter`，统一明细重量、聚合指标、1RM 与占比展示逻辑。
* 新增测试覆盖：新增 `CorePayloadParser`、`DisplayUnitFormatter`、`WorkoutPresenter` 单元测试与 `MainDispatcherRule`。

### 技术改进/重构 (Changed/Refactor)
* 重构 `WorkoutRepository`/`WorkoutPresenter`/`WorkoutUiState`，从“测试数据 ingest”流程切换为“目录导入 + 归档导入导出 + 周期查询 + 报告预览”流程。
* 重构 PR 数据契约，从旧字段切换到标准化字段：`max_weight_kg`、`original_unit`、`original_weight_value`、`estimated_1rm_*_kg`。
* 重构 Android Native 构建：在 `app/src/main/cpp/CMakeLists.txt` 通过 `FetchContent` 引入 `miniz` 与 `toml++`，并接入对应 include/link。
* 调整 Release 构建策略：启用 `isMinifyEnabled`、`isShrinkResources`，并限制 `ndk.abiFilters` 为 `arm64-v8a`。

### 修复 (Fixed)
* 修复 release 混淆下 JNI 入口与返回模型可能被裁剪问题，在 `proguard-rules.pro` 中保留 `WorkoutNativeBridge` 与 `NativeResult`。
* 修复单位切换后的预览一致性问题：切换 `display_unit` 时同步刷新已展开 PR 文本并清理报告缓存，避免旧单位内容残留。

### 弃用/删除 (Deprecated/Removed)
* 删除 Android 侧旧测试资产 `app/src/main/assets/2025_7.txt`，不再依赖单文件内置 ingest。
* 删除旧映射配置 `app/src/main/assets/mapping.json`，迁移为 `app/src/main/assets/mapping.toml`。
* 移除 Insert 页面旧入口 “Ingest test/data/records/2025_7”，替换为导入目录/归档导入导出操作。
