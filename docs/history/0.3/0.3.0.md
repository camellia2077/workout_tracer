## [v0.3.0] - 2026-02-22

### Added
*   新增 **可复用核心库** `workout_core`: 核心业务以独立 target 构建，并通过 C ABI 暴露统一入口 (`workout_core_abi`)。
*   新增 **核心结果模型**: 引入 **`UseCaseResult<T>`** (核心内部) 与 C ABI 结果结构 (平台边界) 以及 `core_error_code`。
*   新增 **核心指标域服务**: 增加 **`training_metrics_service`**，统一 PR/e1RM/频率/密度/分布计算公式。
*   新增 **文本解析入口**: 解析器支持 `ParseText`，并与文件解析共享同一 `istream` 解析流程。

### Changed/Refactor
*   完成 **Phase 1/2/3 核心重构落地**: 核心业务逻辑与表现层/IO 层解耦，形成 `presentation(JSON) -> adapter -> core(C ABI)` 边界。
*   重构 **核心端口设计**: 保留业务端口 (`IMappingProvider` / `IWorkoutRepository` / `IReportExporter` / `IClock`)，移除 `IConsole`。
*   重构 **CLI 调用链路**: CLI 主流程改为经 C ABI 调用核心；命令层仅负责参数解析、JSON/ABI 组装与结果渲染。
*   重构 **请求契约**: 将路径耦合从 `workout_core_request_t` 中剥离，改为 `request_json_utf8` 通道并在 JSON->ABI adapter 处理路径/筛选参数。
*   完成 **目录与构建收口 (Phase 4 已落地部分)**: 代码归拢到 `apps/workout_core/src` 与 `apps/workout_windows_cli/src`，并同步 CMake、脚本与测试路径到 `apps`。

### Fixed
*   修复 **测试产物落盘流程**: 测试流程可正确生成并同步 Markdown 报表到 `test/output/workout_calculator/artifacts`。
*   修复 **脚本路径兼容性**: CLI 脚本迁移后补齐路径、Python 与 CMake 定位逻辑，保证在新目录结构下可用。
