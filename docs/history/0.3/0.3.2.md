## [v0.3.2] - 2026-04-02

### 新增功能 (Added)
* 新增 `weight_kg`、`original_unit`、`original_weight_value` 三段式重量语义，统一承载标准化计算值与原始显示语义。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/workout_core/src/domain/models/workout_item.hpp` 与 `apps/workout_core/src/domain/services/volume_service.cpp` 的组数据模型，将内部重量口径从单一 `weight_` 改为基于 `kg` 的标准化模型。
* 重构 `apps/workout_io/src/infrastructure/converter/log_parser.cpp` 与 `apps/workout_io/src/infrastructure/validation/validator.cpp` 的文本解析规则，将无单位后缀统一视为 `kg`，并将 `l`、`lb`、`lbs` 统一规范为 `lb` 后再换算为 `weight_kg`。
* 重构 `apps/workout_io/src/infrastructure/serializer/serializer.cpp`、`apps/workout_io/src/infrastructure/persistence/manager/db_manager.cpp` 与 `apps/workout_io/src/infrastructure/persistence/inserter/data_inserter.cpp` 的 JSON 与数据库契约，将 `training_sets` 存储模型切换为 `weight_kg`、`original_unit`、`original_weight_value`。
* 重构 `apps/workout_io/src/infrastructure/persistence/facade/query_facade.cpp`、`apps/workout_io/src/infrastructure/reporting/database/database_manager.cpp` 与 `apps/workout_io/src/infrastructure/reporting/formatter/markdown_formatter.cpp` 的查询与报表语义，统一按标准化 `kg` 值计算容量、平均强度与 e1RM，并为聚合结果补充 `common_original_unit` 判定。

### 修复 (Fixed)
* 修复辅助/减重训练记录的容量计算口径，将 `weight_kg <= 0` 的训练组容量固定为 `0`。
* 修复个人记录查询在同重量场景下的非稳定选取问题，改为按 `weight_kg`、`reps`、`date` 的固定优先级选中最终 PR。

### 弃用/删除 (Deprecated/Removed)
* 删除中间 JSON 与 `training_sets` 中旧的 `weight`、`unit`、`elastic_band`、`elastic_band_weight` 语义。
* 删除旧数据库 schema 的兼容扩展路径，旧库结构不再纳入当前版本兼容范围。
